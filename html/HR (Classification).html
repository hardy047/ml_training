Note¶

Assign cost to False Negative and False Positive. Like in this case False Negative is 3 times costlier than False Positive. Because if person is Identify as not leaving company but actualy left new hiring process is costlier. So total cost to company due to false prediction is = [3(FN) + FP] = 372 Now we have to reduce this cost

Finding optimal cut off¶

In [152]:
FP_COST = 1.0
FN_COST = 3.0
In [153]:
def get_cost(cutoff):
    y_knn_v1['new_predict'] = y_knn_v1.prob.map(lambda x: 1 if x > cutoff else 0)
    cm = confusion_matrix(y_knn_v1.actual, y_knn_v1.new_predict, [1,0])
    cost = FN_COST * cm[0][1] + FP_COST * cm[1][0]
    return cost

Notes¶

Classification Model accuraccy: Precesion, Recall, F1-score
Precesion = (TP/TP + FP)
Recall/sensitivity = (TP/TP+FN)
Depends of what qundrant is important to business use Precesion or Recall
F1 Score = (2PR/P+R)
Use F1 if both qundrant is important

Recall or sensitivity gives us information about a model’s performance on false negatives (incorrect prediction of customers who will default), while precision gives us information of the model’s performance of false positives. Based on what is predicted, precision or recall might be more critical for a model.

The cost function comes into play in deciding which of the incorrect predictions can be more detrimental — the false positive or the false negative (in other words, which performance measure is important — precision or recall). https://towardsdatascience.com/model-performance-cost-functions-for-classification-models-a7b1b00ba60
